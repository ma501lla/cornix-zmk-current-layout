# コンボリファレンス

`config/cornix.keymap` で定義されている全コンボの一覧です。

各コンボはデフォルトで `require-prior-idle-ms = <150>` を設定しています。直前のキー入力から 150 ms 経過していないと発動しないため、高速タイピング中の誤発火を防止します。

## 設計思想

このキーマップは **コンボ中心設計**です。**ホームロウ Mods (HM) は採用せず**、修飾キーはすべてコンボでアクセスします。これによりタップ判定のタイミング待ち（いわゆる「ブレーキ感」）と HM の誤発火リスクを根本から排除しています。

修飾コンボは `&sk` (sticky key) ではなく `&kp LXXX` (直接キープレス) を使用しています。理由は以下の通り：

- **保持継続**: コンボを押し続けている間は修飾キーがホールドされ、複数の target キーに連続適用できる
- **2 連続押しの検出**: コンボを素早く 2 回タップすると `Ctrl-Ctrl` のように IDE 機能で検出可能
- **多重修飾の保持**: 複数のコンボを同時保持すれば `Ctrl + Alt + target` のような操作が自然にできる

## 修飾コンボ

2 つのキーを同時押しで該当の修飾キーが発動します。修飾キーはコンボのいずれかが物理的に押されている間保持されます。

### 左手

| 修飾キー | キー位置 | 指 |
|---------|---------|-----|
| L-Shift | A + S (13+14) | 小指 + 薬指 |
| L-Ctrl  | D + F (15+16) | 中指 + 人差し指 |
| L-Alt   | A + D (13+15) | 小指 + 中指 |
| L-GUI   | Z + X (25+26) | 底行 小指 + 薬指 |

### 右手

| 修飾キー | キー位置 | 指 |
|---------|---------|-----|
| R-Shift | U + I (7+8) | 人差し指 + 中指 |
| R-Ctrl  | O + P (9+10) | 薬指 + 小指 |
| R-Alt   | M + , (33+34) | 底行 人差し指 + 中指 |
| R-GUI   | , + . (34+35) | 底行 中指 + 薬指 |

### 多重修飾の組み合わせ

- **同手 Shift + Ctrl** (例: Ctrl+Shift+Z): ホームロウ 4 キー同時押し (左 A+S+D+F、右 U+I+O+P)
- **対角多重修飾** (例: Ctrl+Alt): 左右のコンボを 1 つずつ同時保持 (例: D+F + M+,)
- **同手 Shift+Alt / Ctrl+Alt 等**: 共通のキーを使うため対角で実現する必要あり

| コンボ  | キー位置 | 動作 | 補足 |
|--------|---------|------|------|

| IME 切替 | E + R (3+4) | `&my_macro_zenhan` | Alt + `` ` `` を発射 (Windows IME) |

### mod-morph による Delete アクセス

**Delete 専用のコンボは設けていません**。Shift コンボと Backspace コンボを同時に押すと Delete が発射されます：

| 操作 | 結果 |
|------|------|
| I + O 単独 | Backspace |
| (A+S) + (I+O) | Delete (Shift + Backspace が DEL に morph) |
| (U+I) + (I+O) | Delete |

この仕組みは `config/cornix.keymap` 内の `bs_del` ZMK mod-morph 振る舞いで実現しています。

## 括弧・記号

**コンボではなくなりました** — **Symbols レイヤー**上の通常キー (上段と home 行) に配置しています。

| 記号 | Symbols レイヤー上の位置 | 物理キー |
|------|---------------------------|---------|
| `<`  | top 行 pos 1 | Q |
| `>`  | top 行 pos 2 | W |
| `[`  | top 行 pos 3 | E |
| `]`  | top 行 pos 4 | R |
| `{`  | top 行 pos 5 | T |
| `}`  | top 行 pos 6 | Y |
| `(`  | home 行 pos 21 | L (Shift+9 と等価) |
| `)`  | home 行 pos 22 | ; (Shift+0 と等価) |

使い方: 親指 `mo 1` (位置 42) を hold して Symbols を起動し、対応するキーを押す。

旧 vertical pair コンボ (例: T+G で `{`、R+F で `(`) は打鍵時の操作感が悪く撤去しました。

## レイヤー起動コンボ

レイヤーをモーメンタリで起動します。ホールド型レイヤーキーと同じ動作。

| コンボ | キー位置 | レイヤー | Timeout |
|-------|---------|---------|---------|
| → Symbols (mo 1) | C + X (27+26) | Symbols | 60 ms |
| → Symbols (mo 1) | , + . (34+35) | Symbols | 60 ms |
| → Symbols (mo 1) | F + Z (16+25) | Symbols | 60 ms |
| → Mix (mo 2) | J + / (19+36) | Mix | 60 ms |

> 注: `, + .` は R-GUI コンボと重複しています。押し方のタイミングによりどちらが発動するかが変わるため、衝突が頻発する場合は位置を見直してください。

## 編集に関する注意

- **ZMK Studio** はレイアウトの閲覧と静的キーマップ編集に使えますが、`bs_del` `hm` レイヤータップなどの**振る舞い (behavior) 定義は編集できません**。これらは `config/cornix.keymap` を手で編集して再ビルドする必要があります
- **Keymap Editor** は推奨されません。過去にコンボの key-positions を保存時に勝手に書き換える挙動が確認されており、不整合の原因となります
